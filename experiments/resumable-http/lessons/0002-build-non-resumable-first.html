Build non-resumable first

Your repo is empty enough. Good. Next: prove plain streaming before resume.

Goal

In code-examples/tiny-typescript-demo/index.ts, build one endpoint:

GET /stream
→ sends 10 server-sent events
→ one event every 500ms
→ then closes

This teaches the wire format. Resume comes after you can see plain stream bytes.

Rules

  • Use Node built-in node:http. No Express. No framework yet.
  • Use SSE framing: id: line, data: line, blank line.
  • Use env.PORT from your existing env.ts.
  • Keep it under ~70 lines.

Shape to aim for

import http from "node:http";
import { env } from "./env.ts";

const server = http.createServer(async (req, res) => {
  if (req.url !== "/stream") {
    res.writeHead(404).end("not found\n");
    return;
  }

  res.writeHead(200, {
    "content-type": "text/event-stream",
    "cache-control": "no-cache",
    connection: "keep-alive",
  });

  // your loop here
});

server.listen(env.PORT, env.HOST, () => {
  console.log(`http://${env.HOST}:${env.PORT}`);
});

Self-check

Run server:

cd code-examples/tiny-typescript-demo
pnpm dev

In another terminal:

curl -N http://localhost:8086/stream

Expected rhythm: one event appears every half second:

id: 1
data: {"delta":"token-1"}

id: 2
data: {"delta":"token-2"}

Before moving on, answer from memory

  1. Why is there a blank line after each event?
  2. Where will the resume cursor live later?
  3. If curl disconnects, should this first version keep producing?
