Split producer from listener

Current code streams directly inside the request. Good first step. Resumable streams need one extra split.

New mental model

producer = creates events over time
listener = one HTTP response receiving events
run      = event log + current listeners

If listener dies, producer continues. That is the whole magic behind refresh-safe streams.

Target API

POST /runs
  create run
  start producer
  attach this response as first listener

GET /runs/:id/events?after=3
  replay stored events with id > 3
  attach this response for future events

Types to add

type StreamEvent = {
  id: number;
  data: { delta: string };
};

type Run = {
  id: string;
  nextEventId: number;
  events: StreamEvent[];
  clients: Set<http.ServerResponse>;
  done: boolean;
};

const runs = new Map<string, Run>();

Functions to write

1. writeEvent(res, event)

res.write(`id: ${event.id}\n`);
res.write(`data: ${JSON.stringify(event.data)}\n\n`);

2. emit(run, data)

  • create event with run.nextEventId++
  • push into run.events
  • write to every run.clients

3. attach(req, res, run, after)

  • write SSE headers
  • replay matching events with for (const event of run.events) if (event.id > after) writeEvent(res, event)
  • if done, end response
  • else add response to run.clients
  • on request close, remove response from run.clients

4. startProducer(run)

  • loop 10 times
  • call emit(run, { delta: `token-${i}` })
  • sleep 500ms
  • mark done
  • end all current clients
