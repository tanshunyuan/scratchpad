Electron Preview Architecture

Goal: add right-side live preview to Electron coding app. Not cloud IDE. Not Replit clone.

Core model

React editor UI
  → writes project files
  → Electron main spawns dev server
  → preview pane loads http://localhost:PORT
  → dev server HMR updates preview

Local MVP pieces

Editor

Monaco or existing editor. Saves file content to project folder.

Runner

Electron main uses Node child_process.spawn() to run npm run dev.

Preview

WebContentsView or iframe loads local dev-server URL.

Logs

Pipe child process stdout/stderr to renderer over IPC.

Env variables

spawn('npm', ['run', 'dev'], {
  cwd: projectDir,
  env: {
    ...process.env,
    ...projectEnv
  }
})

Secrets belong in main process/storage, not React renderer. Only expose public frontend env intentionally.

When Docker enters

  • User runs untrusted code.
  • Need reproducible Linux env.
  • Need dependency isolation per project.
  • Need port/process cleanup guarantees.
Electron → Node backend → Docker container → app port → preview URL

Security defaults

  • Keep preview isolated from app UI.
  • Do not enable Node integration in preview.
  • Use context isolation.
  • Treat preview content as untrusted if user code can run arbitrary JS.
