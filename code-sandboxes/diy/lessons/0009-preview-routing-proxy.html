Preview routing proxy

Mission: make remote sandbox preview reachable by iframe through a routed URL, not container localhost.

Tangible win: after this lesson, you should explain why remote preview needs a proxy route.

Problem

Local runner can return:

http://localhost:5174

In remote sandbox, that breaks. In Electron iframe, localhost means the user's machine, not remote container.

Electron on user laptop
→ iframe loads http://localhost:5174
→ tries user laptop port 5174
→ remote container never reached

One idea

Remote preview needs a public routed URL.

Electron iframe
→ https://preview-abc.yourapp.com
→ preview proxy
→ sandbox container :5174

The proxy is remote version of Docker port publishing.

Local Docker:
-p 5174:5174

Remote sandbox:
preview URL → proxy → container:5174

What proxy knows

sandboxId: abc
preview host/path: /preview/abc
container target: http://10.0.4.21:5174

When browser requests preview URL, proxy looks up sandbox target and forwards request.

Two URL styles

Subdomain style

https://abc.preview.yourapp.com

Good isolation story later. Needs wildcard DNS/TLS.

Path style

https://runner.yourapp.com/preview/abc

Easier to prototype. Some dev servers need base-path handling.

Local fake for learning

You can simulate remote proxy locally:

runner service: http://localhost:8086
preview route:  http://localhost:8086/preview/:sandboxId
actual app:     http://localhost:5174

Electron iframe stops using localhost:5174 directly. It uses runner preview route.

POST /sandboxes/start
→ starts app on localhost:5174
→ returns previewUrl: http://localhost:8086/preview/abc

Then runner proxies:

GET /preview/abc/*
→ forward to http://localhost:5174/*

Why this matters

  • Electron only needs a URL.
  • Runner controls which sandbox URL maps to which container.
  • Remote containers can stay private.
  • Auth/access control can live at proxy boundary later.

What not to build yet

  • No wildcard DNS.
  • No TLS automation.
  • No multi-tenant auth.
  • No production reverse proxy.

Prototype path-based proxy first.

Retrieval check

Question: why is http://localhost:5174 wrong for real remote sandbox?

Next tiny build step

Add a local fake preview route to the runner:

GET /preview/:sandboxId/*
→ find sandbox
→ forward request to sandbox.previewUrl

Then change POST /sandboxes/start to return:

previewUrl: "http://localhost:8086/preview/<sandboxId>"

Keep actual Vite app on localhost:5174 for now. Goal is routing boundary.
