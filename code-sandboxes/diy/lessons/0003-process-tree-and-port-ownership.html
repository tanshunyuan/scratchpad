Process tree and port ownership

Mission: make preview stop/restart reliable when a dev server has child processes.

Tangible win: after this lesson, you should know why “process exited” can still leave a port busy.

Where you are now

You have a working lifecycle:

start → spawn → wait for URL → running
stop → SIGTERM → wait for exit → stopped
restart → stop complete → start again

This is correct for one process. But dev servers are often not one process.

One idea

The process you spawn is not always the process that owns the port.

Electron main
└─ shell              ← created by shell: true
   └─ pnpm dev        ← package manager wrapper
      └─ vite         ← often the process listening on :5174

Your variable devServerProcess may point at the shell or package-manager wrapper. The actual server process may be deeper in the tree.

Why this matters

If you kill only the parent process, a child can sometimes survive. Then restart fails because old Vite still owns the port.

restart
→ kill parent
→ parent exits
→ child vite survives
→ new vite starts
→ port 5174 already in use

This is why “I waited for exit” can still be insufficient in process-tree cases.

Port owner beats process handle

Preview readiness asks: “Does URL respond?”

Preview cleanup asks: “Is anything still listening on port?”

lsof -i :5174

On macOS/Linux, this shows which process owns the port. If restart is weird, this is first debug command.

Three simple strategies

1. Avoid shell when possible

shell: true adds a wrapper process. If you do not need shell features, prefer direct spawn.

spawn('pnpm', ['dev', '--port', String(previewPort)], {
  cwd: projectDir,
  env: process.env
})

Fewer wrappers means easier lifecycle. Source: Node child_process spawn docs.

2. Treat port-busy as cleanup signal

If start fails because port is busy, do not guess. Inspect port owner.

lsof -i :5174
ps -p <PID> -o pid,ppid,command

3. Use tree-kill only when needed

If child processes survive often, add process-tree cleanup. Keep it as later infra, not first MVP.

Electron → stop preview → terminate process tree → verify port free → start again

Next tiny build step

1. Start preview.
2. Run: lsof -i :5174
3. Stop preview.
4. Run: lsof -i :5174 again.
5. If no output, cleanup is good enough for now.
6. If process remains, next fix is process-tree cleanup.
